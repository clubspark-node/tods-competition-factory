---
title: Print Policy
---

The **Print Policy** (`POLICY_TYPE_PRINT`) is a tournament-level
override slot for printed-artifact composition: how draw sheets,
order-of-play sheets, player lists, court cards, sign-in sheets, and
match cards should look when generated by `pdf-factory`. The policy is
intentionally opaque to the factory — its full shape lives in
`pdf-factory`'s `CompositionConfig` type, validated by the editor and
consumed by the print dispatcher at PDF-generation time.

**Policy Type:** `print`

**When to Use:**

- Per-tournament tweaks to print appearance that override the
  provider-scoped default (different header layout, custom alert
  banner, alternate cell density, etc.) for one specific tournament.
- Capturing one-off art direction (sponsor logos in the header for a
  single event, a particular surface label for a regional final, etc.)
  without touching the provider's standing print configuration.

For the _provider-scoped default_ — the per-print-type composition
that applies to every tournament under a given provider — the policy
lives outside the factory in
`competition-factory-server`'s `providerConfigSettings.policies.printPolicies`
and is authored via the admin client. The factory's
`POLICY_TYPE_PRINT` extension is _only_ the per-tournament override
layer.

---

## Resolution Order at Print Time

`pdf-factory.resolveCompositionConfig({ providerConfig, tournamentRecord, printType, runtime })`
deep-merges three layers, with later layers winning per-field:

1. **Provider default** — read from
   `providerConfig.providerConfigSettings.policies.printPolicies[printType]`.
2. **Tournament override** — read from the tournament record's
   extension where `name === 'print'`. The extension `value` is keyed
   by `printType`.
3. **Runtime tweaks** — modal-time edits (a one-print-only checkbox
   the user just toggled) passed by the caller.

The result is passed to `executePrint(...)` which routes to the
correct generator.

## Policy Structure

The factory treats this policy as opaque JSON. The shape below is
defined by `pdf-factory`'s `CompositionConfig` and serves as a
reference for tooling that authors the policy.

```ts
{
  print: {
    // Sparse — every nested field is optional. Unset fields fall
    // through to the next resolution layer (or pdf-factory defaults
    // if no layer sets them).
    [printType in 'draw' | 'schedule' | 'playerList' | 'courtCard' | 'signInSheet' | 'matchCard']?: {
      name?: string;
      page?: {
        pageSize?: 'a4' | 'letter';
        orientation?: 'portrait' | 'landscape' | 'auto';
        margins?: { top: number; right: number; bottom: number; left: number };
      };
      header?: {
        layout?: 'grand-slam' | 'itf' | 'minimal' | 'wta-tour' | 'national-federation' | 'none';
        tournamentName?: string;
        subtitle?: string;
        // …additional fields per HeaderConfig
      };
      footer?: {
        layout?: 'standard' | 'seedings' | 'officials' | 'none' | 'seedings-table' | 'prize-money' | 'officials-signoff' | 'combined-tour';
        showTimestamp?: boolean;
        showPageNumbers?: boolean;
        // …additional fields per FooterConfig
      };
      content?: {
        draw?: {
          includeSeedings?: boolean;
          includeScores?: boolean;
          splitStrategy?: 'single-page' | 'halves' | 'quarters';
          showDrawPositions?: boolean;
          showByes?: boolean;
        };
        schedule?: {
          cellStyle?: 'detailed' | 'compact';
          showMatchNumbers?: boolean;
          alertBanner?: string;
          showPotentialParticipants?: boolean;
        };
        // …playerList, courtCard, etc.
      };
    };
  };
}
```

## Default Fixture

`POLICY_PRINT_DEFAULT` (in `src/fixtures/policies/POLICY_PRINT_DEFAULT.ts`)
is intentionally minimal — an empty override that defers entirely to
the provider default. It exists so callers can attach a no-op
extension as a baseline before a tournament admin layers customizations
on top.

```ts
import { POLICY_PRINT_DEFAULT } from 'tods-competition-factory';

POLICY_PRINT_DEFAULT;
// → { print: { policyName: 'Default Print Configuration' } }
```

## Attaching to a Tournament

```ts
import { tournamentEngine } from 'tods-competition-factory';

tournamentEngine.attachPolicies({
  policyDefinitions: {
    print: {
      policyName: 'French Open 2026 — print overrides',
      schedule: {
        header: { layout: 'grand-slam', tournamentName: 'Roland Garros' },
        content: { schedule: { alertBanner: 'Centre Court closes at 23:00' } },
      },
    },
  },
});
```

The tournament-level extension overrides only the fields it sets;
everything else inherits from the provider default (or pdf-factory
hardcoded fallback if no provider policy exists).
